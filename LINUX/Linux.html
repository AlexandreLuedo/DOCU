2.2. Fichiers

2.2.1. Types

  1. Ordinaires
    • aussi appelés réguliers (ordinary, regulare files)
    • img, texte, son, programme executable
  2. Répertoire
    • structure l’espace disque en arborescence
    • contient une table identifiant nom et numéro interne (numéro inode)
  3. Spécial
    • /dev
    • peuvent être crée partout avec mknode
    1. bloc
      • accéde à des blocs de données (disque dure)
    2. caractère
      • accède phériphérique octet par octet (écran de terminal, lecteur de bandes)
    3. FIFO
      • First In First Out
      • tubes (pipes), canaux de communications d’échange de données entres processus
    4. socket
      • permet également échange de données entre processus mais via protocole réseau classique
    5. liens symboliques
      • fichiers renvoient à d’autres fichiers, contenant interne: chemin d’accès du fichier cible

2.2.3. Caractères génériques / jokers

Permet de désigner/afficher selon certains critères

caractère rôle
* tout
? caractère unique
[…] un seul caractère appartenant à la liste unique
[a-b] entre a-b
[!…] un seul n’appartenant pas à la liste
[^…] un seul n’appartenant pas à la liste
[x,y] liste éléments, chiffres, n’importe
[x..y] suite d’éléments incrémentés de x à y
  1. Exemples
    echo ???? # touts les fichiers contenants uniquement 4 caractères
echo b????* # touts les fichiers commençant par la lettre b de au moins 5 caractères
echo [bd]* #tout les fichiers commençant par b ou d
echo [!bd]* # tout les fichiers ne commençant pas ni par b ni par d
echo [^bd]* # "
echo *{df,c2,3}* # tout les fichiers du répertoire courant qui où le nom contient df ou c2 ou 3
echo ~/.* # tout les fichiers commençant par un point

2.6.2. egrep

recherche plus étendue équivaut à grep -E

  1. Critères de recherche
    <pipe> Ou logique, l’expression située avant ou après doit apparaître
    (…) Groupement de caractères
    […] Un caractère à cette position parmis ceux indiqués
    . Un caractère quelconque.
    + Répétition, le caractère placé avant doti apparaître au moins une fois.
    * Répétition, le caractère situé avant doit apparaître de zéro à n fois
    ? Le caractère situé avant doit apparaître exactement n fois
    {n} le caractère situé avant doit apparaître n nombre de fois
    {n, } il apparaît n fois ou plus.
    {n,m} il apparaît entre n et m fois
    ^ en début de chaîne
    $ en fin de chaîne

2.6.3. fgrep

  • Simplifié et plus rapide grep -F
  • Ne gère pas les expressions régulières, recherche que des chaînes de caractère.

2.6.4. sed

  • Editeur de flux permettant de filtrer et de transformer du text. Utilise quasimenent les mêmes commandes que vim.

s/ancien/nouveau/[g]

  • Le g optionnel final permet de remplacer tout les occurences.
$ echo "Je m'appelle _nom_, Jean _nom_." | sed -e 's/_nom_/Durand/'
je m'appelle durand, Jean _nom_. # manque le g

$ echo "Je m'appelle _nom_, Jean _nom_." | sed -e 's/_nom_/Durand/g'
je m'appelle durand, Jean Durand.
  • Supprimer les lignes vides (en utilisant la syntax vim)
$ cat fichier
Ligne 1
Ligne 2

Ligne 3


FIN
$ sed -e '/^ *$/d' fichier
Ligne 1
Ligne 2
Ligne 3
FIN
  • Argumentaires de commande
'/^ *$/d' # suppr tout les espaces d'une ligne
'^' #selectionner le début de la ligne
' *' #un ou plusieurs espaces
'$' #fin de ligne
'//' #servent à délimiter la zone d'action

2.7.1. Attributs

  1. PID
    • Process ID
    • Numéro unique du processus.
    1. Exemple
      • Le premier processus créer par le noyau avec PID 1 est soit init ou systemd.
  2. PPID
    • Parent Processus ID
    • Processus toujours créer par un autre (sauf le processus initial créer par le noyau), via l’appel fork
  3. UID & GID
    • User ID
    • Group ID
    • Portent ensemble les mêmes droits que le group principal de l’appareil. Ils donnent les droits pour tous les utilisateurs, determinent les chemins d’accès aux objets du système (fichiers, répertoire etc).
  4. Duré de traitement & priorité
    • Correspond entre le temps du réveil du processeur et celui du processus.
    • Dans un environement multitâche
      • Les processus de haute prioritée sont executés d’abord
      • Ensuite, les processus inactifs sont d’avantage prioritaire par apport à ceux déjà executés.
    • ordonnateurs des tâche qui gère les priorités et les temps d’éxecution.
  5. Répertoire courant
    • Repertoire depuis lequel le processus a été lancé.
    • Est transmis au processus. Servira de base pour les chemins relatifs.
  6. Fichiers ouverts
    • trois descripteurs sont asssociés à des fichiers: 0, 1, 2 (entrées/sorties standards). A chaques ouverture de fichier ou de nouveau canal, la table se remplit.
    • Fermeture, ces fichiers sont fermés.

3.3. Système de fichiers

  • Les ext*, XFS, BTRFS, exFAT, etc… sont des systèmes de fichiers, et donc des types de formatages de périphériques de stockages.

3.3.1. Métadonnées

  • Elles sont stockés dans un fichier appelé inode.

3.3.2. ext*

  • Système de fichiers utilisés avec linux.
  1. ext2
    • Non journalisé, donc peux corrompre les fichiers en écriture si arrêt brutale de celle-ci. Peu utilisé.
    • Limites d’écriture: 2To par fichier, sur 32To en tout.
  2. ext3
    • Optionnellement journalisé, utilisé pour la compatibilité entre ext2 et ext3.
  3. ext4
    • Destiné à remplacer le ext3. Gère du 16To par fichier, sur 1Eo. Dates et fichiers peuvent être gérées jusqu’en 2514.
    • Compatible avec ext3 en lecture & écriture.
    • Système par défaut de nombreuses distros, surtout Debian.

3.4. Partitionnement

3.4.1. Méthodes

  • MBR -> pour machines sous BIOS
  • GPT -> UEFI, BIOS
  1. MBR & BIOS
    • Codé sur 32bits, taille max par partition est de 2To.
    • Possibilitée d’étendre sous plusieurs disques dures le système de fichier.
  2. MBR
    • Premier secteur disque, est un MBR (Master Boot Record). Il contient une zone d’amorce. Contenant 512 octets.
  3. Les partitions
    • IDE = 63 partitions par disque.
    • SCSI = 15.
    • libata = 15 pour tout types de disques (pour tout les linux utilisant l’API libata).
    • Possibilité de dépasser les 15 partitions par disques grâce au device mapper de Linux, (appelant LVM (Logical Volume Management))
    1. Numérotation
      • hda1 première partition primaire du disque IDE
      • hda5 cinquième partition, premier lecteur logique du second disque IDE
      • sda3 troisième partition primaire du premier disque SCSI/libata
      • sda8 huitième partition, soit quatrième partition logique du troisième disque SCSI/libata
    2. Sortes
      1. Primaires
        • 4max. par disques, (ce sont celles du MBR).
      2. Étendues
        • Primaires étendues, une seul par disque.
      3. Logiques
        • Créés à l’intérieur d’une partition étendue.
  4. Types de partitions
    • Toutes partitions sont nommé en hexadecimal, Linux reconnaît généralement le contenu d’une partition par le système de fichiers qui y réside.
    • Type 0x83 (Linux) peut acceuillir tout type de système de fichiers Linux:
      • ext2
      • ext3
      • ext4
      • jfs
      • XFS
      • etc…
  5. Partitionnement GPT
    • GPT (GUID Partition Table)
    • GUID (Globally Unique Identifier) est un standart permettant de décrire la table de partitionnement d’un disque dur, faisant partie des spécifications de UEFI (Unified Extensible Firmware Interface)
    • UEFI remplace le BIOS
    • GPT remplace le partitionnement MBR
  6. GPT et UEFI
    • Initié par Intel, toutes les cartes mères sous UEFI tournent GPT.
    • L’on peux utiliser GPT avec certains BIOS, via partition spéciale comme la BIOS Boot partition couplé à un gestionnaire de démarrage particulier, comme GRUB2.
    • L’adresse des blocs logiques est codée sur 64 bits.
    • Toutes les versions courantes de linux utilisent le partitionnement GPT.
  7. GUID
    • C’est un identificateur unique, une valeur codée sur 128 bits servant d’identificateur unique à pour un composant logiciel.
    • Voici ses ensembles:
      • Un bloc de quatre octets
      • Trois blocs de deux octets
      • Un bloc de siw octets.

    • Dans GPT le GUID sert à décrire

      • L’identifiant unique de disque
      • L’identifiant de type de partition (équilvalent au types de partitions MBR).
      • L’identifiant unique d’une partition
      E4A1B2C3-D4F5-6789-ABC0-123456789ABC
  8. LBA 0
    • le LBA est les 512 premiers octets du disque. Conserve son rôle de MBR, appelé Protective MBR. Il est la signature des partitions GPT.
  9. LBA 1
    • L’en-tête de GPT en position LBA 1.
    • Une copie existe en LBA -1 qui est en partant de la fin du disque.
    • Contient les informations concernant la structure GPT.
    • Son emplacement et sa taille.
    • L’emplacement et la taille de l’en-tête secondaire (à la fin du disque)
    • les sommes de contrôle associées.
    • la première et la dernière adresses utilisables pour les paritions.
    • le GUID du disque
    • l’adresse de la table des partitions (normalement LBA 2)
    • le nombre et la taille de descripteurs de partitions
    • la somme de contrôle de la table des partitions.

    L’ensemble occupe 92 octets, les 420 restants sont remplis de zéros. En cas d’erreurs sur le LBA 1 le LBA -1 prend le relais. En cas d’erreurs sur les deux, le disque devient inutilisable. Pareil pour le contrôle des tables de partitions

  10. LBA 2 à 33
    • Contiennent des descripteurs de partitions.
    • Chaque partitions est écrite sur 128octets soit 4 par LBA.
    • Les 32 LBA permettent d’écrire 128 partitions. C’est le minimum imposé par le standard UEFI, le nombre, la taille de chaques partitions peuvent être modifié dans le LBA 1.
    • Une sauvegarde est présente du LBA -2 à LBA -33 (fin de disque).

  11. UEFI Boot manager
    • UEFI remplace BIOS, et est stocké dans le firmware de l’ordinateur. Sa config se situe à deux endroits:
      • La NVRAM (Non Volatile RAM), équivalent du CMOS des anciens BIOS, à la différence que son contenu est normalisé.
      • La partition ESP
      • Comme il n’y a plus de MBR il faut savoir identifier les supports & systèmes amorçables (l’équivalent d’un menu de démarrage). Donc il y a un menu graphique qui permet de selectionner le système à démarrer.
      • Ce que la NVRAM fait:
        • Quel système est actuellement démarré
        • L’ordre d’entrée de boot
        • Le timeout avant de booter sur l’entrée par défaut.
        • La liste des entrées (nom, support (disque), fichier à démarrer depuis la partition ESP)
  12. Partition système EFI
    • UEFI exécute le fichier indiqué dans les options de boot.
    • Ils sont stocké sur une partition spéciale appelé ESP (EFI System Partition).
    • Elle contient les différents informations liées aux démarrage de chaques OS, de la config d’un chargeur de démarrage, parfois, d’un chargeur de démarrage comme GRUB.
    • Elle se compose d’une arborescence de répertoire.

3.4.3. Manipuler les systèmes de fichiers

  1. Définition

    Un système de fichiers gère des blocs de données dans un espace de stocdkage. Il organise les données dans des fichiers, stockés dans une arborescence structurée par des répertoires. Des structures permettent de gérer les attributs généraux de l’espace de stockage du système de fichiers. Remarque: L’espace de stockage peut être physique (disque, partition) ou logique (volume logique VLM). Il existe également des systèmes de fichiers virtuels, gérés en mémoire vive.

  2. Bloc

    Unité de base, atomique, de stockage du système de fichiers. Un fichier occupe toujours un nombre entier de blocs. Ainsi, si un fichier ne contient qu’un seul osctet et qu’un bloc possède une taille de 4096 octets, 4095 octets sont inutilisés.
    Remarque:
    Certaines commandes affichent la taille des fichiers en blocs, comme du, df, ou find.
    La taille d’un bloc est de 512 octets poru certaines commandes, 1024 pour d’autres.

  3. superbloc
    • Chaques systèmes de fichiers linux possèdent au moins un superbloc (superblock).
    • Zone de métadonnées conteannt des informations sur le système de fichier et son état.
      • Type de système de fichiers
      • Taille
      • État
      • Position d’autres zones de métadonnées (autres superblocs, table d’inodes, etc…)
  4. Table d’inodes
    • Un inode (index node) c’est un noeud d’attributs. C’est une structure de données contenant les informations d’état d’un fichier que l’on appelle des attributs ou métadonnées. Chaque fichier comporte son numéro d’inode. Ces fichiers sont stockés au sein d’une table, la table des inodes.
    • Une table d’inode fait partie des métadonnées.
    1. Contenu d’un inode
      • Varie d’un système de fichier à un autre.
      • Respect la norme POSIX
      • Contient:
        • Taille
        • Identifiant du périphérique le contenant
        • Propriétaire
        • Groupe d’utilisateurs associé
        • Numéro d’inode
        • Mode: type et droits d’accès
        • Date de ernière modification d’attribut (change time)
        • Date de dernière modification de contenu (modification time)
        • Compteur de liens physiques (hard links): nombre de noms de fichiers associés à l’inode.
        • Adresses vers des blocs de données et/ou des blocs d’adresses de blocs de données.

      La commande stat affiche les attributs de l’inode associé ) un fichier. 

      $ stat /boot/vmlinuz-5.10.0-22-amd64
    2. Adresses
      • Adresses pointant sur les premiers blocs de données du fichier
      • Adresses pointant sur des blocs contenant d’autres champs d’adresses.
        • Dans le cas ou les adresses se pointent entre elles et pointes d’autres adresses etc… l’on appel ça des blocs d’indirections (simple, doubles, triple)
