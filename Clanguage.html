1. Compilation and compiler

When a C programm are send to the compiler, he split the work in tree parts :

1.1. The pre-compilation

The compiler will take the file.c and include all the pre-compilation directives who are include white an # before the declaration. Like #include <stdio.h> and send it the processor.

1.2. The compilation

If the pre-compilation process doesn’t fail, the compiler will start the compilation process. The compiler will translate all the code into machine code (binary code).

1.3. The links editing

For example, in the code we have include the <stdio.h> library, who are compiled and store in another location. So the links editing will make the link between library and the code for giving an code result.

Between the differents tasks, the file aren’t in .c the file is in an intermmediate state. Normally after compiling the compiler will erase this intermmediate files, but you can specify with options to keep this files.

3. Algorithm

3.1. How to

For solving, or writing problems in code, you must follow this differents rules

3.1.1. Clarly define the problem

  • Define all the datas, links, docs etc…
  • Define goals, calculs, and what you will use to resolve the problem.
  • Define all frontiers and what you can’t do

3.1.2. Research an solution method

  1. In first of all you must :
    • Clarify the state “énoncé”
    • Simplify the problem
    • Verifing if the problem is solvable and can be traited.
  2. Search an strategy for the algoritm build.
    • Cut the problem into tiny multiples problems who can be more simple to resolve
    • Make successive refinements
    • The most succesive raffinement possible are the instructions.

3.1.3. creation of an algorithm

  1. Definition

    The algoritm are multiples rules or instructions who are here to solve the problem. He must be construct regardless of the programming language.

  2. Warn
    • The algorithm must be execute with an finish number of operations
    • The algorithm must be clarly speciefed, without embiguitty
    • The data must be precize (integer, real, etc…)
    • The algorithm give at least one result
    • The algorithm must be effective (an human can simulate them with an finish time)
